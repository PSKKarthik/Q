ABSTRACT
QGX (Query Gen X) is a cloud-ready, AI-enabled Learning Management System developed to solve practical academic management challenges in schools and colleges. Many institutions currently depend on disconnected tools for attendance, announcements, assessments, and student tracking, which causes delayed decision-making and repeated manual work. QGX addresses this by providing a unified, role-based platform with secure authentication, structured dashboards, and integrated educational workflows. The system is built with Next.js 14, TypeScript, Supabase (Auth + PostgreSQL + Realtime), and GROQ AI APIs, offering both modern usability and maintainable architecture.

The project supports four primary roles: Admin, Teacher, Student, and Parent. Admin users can perform governance tasks such as user lifecycle management, activity monitoring, and announcements. Teachers can manage assessment operations and AI-assisted content generation. Students receive a guided academic interface for participation and progress tracking. Parents can monitor linked student outcomes. In addition, the platform includes password recovery, audit logging, and Progressive Web App (PWA) features.

From a software engineering perspective, QGX demonstrates full lifecycle execution: requirement analysis, feasibility assessment, architecture and module design, implementation, testing, and deployment-readiness validation. The final system offers a balanced outcome: strong usability for non-technical users, technical correctness for evaluators, and extendability for future research or institutional deployment.

INTRODUCTION
Digital transformation in education is no longer optional. Institutions now require systems that can support secure data handling, role-specific workflows, and real-time visibility without creating complexity for users. Traditional fragmented ecosystems fail to deliver this consistency, especially when the number of users and operations grows. QGX was conceptualized to solve this gap by combining modern full-stack web engineering with practical academic workflows.

The project is intentionally designed for clarity, maintainability, and operational realism. Instead of over-engineering, QGX focuses on foundational institutional needs: secure user access, role-based dashboards, workflow automation, activity traceability, and AI-assisted productivity support.

1.1 Purpose of the System
The purpose of QGX is to provide a centralized and secure digital platform that simplifies teaching, learning, and academic administration.

Detailed purpose points:
1. Replace disconnected educational tools with one integrated system.
2. Minimize repetitive manual operations in administration and communication.
3. Support teachers with AI-assisted workflow acceleration.
4. Improve student interaction quality and visibility of academic progress.
5. Provide audit-friendly transparency for administrators.
6. Improve parent visibility of student-level outcomes.

1.2 Scope of the System
QGX covers major web-based LMS operations for institutional use. It includes user onboarding, secure role-aware access, operational dashboards, AI-assisted features, and quality assurance support.

In-scope:
1. Registration, login, logout, forgot password, and reset password.
2. Role-specific dashboards and protected module access.
3. Admin-level user operations including batch user creation.
4. AI query endpoint integration via secured backend route.
5. Notification and announcement workflows.
6. Activity logging and monitoring.
7. PWA installability and basic offline fallback.
8. Unit, integration, and E2E testing support.

Out-of-scope for current phase:
1. Full institutional ERP integration.
2. Payment and billing modules.
3. Native Android and iOS applications.
4. Deep analytics and ML-driven adaptive recommendation engine.

1.3 Existing System and Limitations
In many institutions, operations are split across spreadsheets, messaging groups, isolated portals, and email threads. This creates poor synchronization, weak accountability, and high manual correction effort. Teachers and administrators often spend more time coordinating operations than improving educational quality.

Limitations observed:
1. Duplicate data entry and inconsistent updates.
2. Delayed communication between stakeholders.
3. Insufficient role-based access protection.
4. Weak traceability of critical operations.
5. Poor mobile usability and no offline fallback.
6. No practical AI assistance in everyday teaching operations.

1.4 Proposed System and Advantages
QGX proposes a role-driven, secure, and modular LMS architecture. The platform centralizes core workflows and adds controlled AI support while maintaining data governance and usability.

Advantages:
1. Centralized source of truth for operations and users.
2. Faster execution of routine institutional workflows.
3. Reduced operational errors through validation-first APIs.
4. Stronger security through layered authentication and authorization.
5. Better user experience across desktop and mobile.
6. Improved maintainability through modular and typed codebase.
7. Real-time visibility via notifications and activity logs.

Flow:
